# Project Development Rules

이 프로젝트의 화면 개발, 백엔드 로직 수정 및 신규 기능 개발 시 항상 다음 두 가지 README 문서를 우선적으로 참고하여 프로젝트의 아키텍처, 기술 스택, 코드 컨벤션, 개발 규칙을 준수해야 합니다.

1. **Frontend Development Guide**
   - 파일 경로: `everportal/README.md`
   - 역할: React 19, TypeScript, Vite 기반의 프론트엔드 개발 시 Schema-driven Validation(react-hook-form + zod) 및 컴포넌트 구성 가이드라인.

2. **Backend Development Guide**
   - 파일 경로: `eversrm/README.md`
   - 역할: Spring Boot 3 기반의 백엔드 개발 시 아키텍처 계층 분리, 패키지 구조, MyBatis 사용 및 코드 작성 컨벤션.

작업 전 반드시 위 파일들에 명시된 사항을 숙지하고, 요구사항 구현 시 해당 규칙을 어기지 않도록 주의하세요.
